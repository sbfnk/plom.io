plom.io/epidemiology

  • A web platform to share and interact in real time with epidemiological models.
  • Based on a formal layered grammar for epidemiological models.
  • Offering a common API to fit epidemiological models to data.

A layered grammar for epidemiological models

http://www.plom.io/doc/modeler/intro

process.json

{
 "meta": {"id": "SI",
          "comment": "SI model"},

 "state": [{"id": "S", "comment": "Susceptible"},
           {"id": "I", "comment": "Infectious"}],

 "parameter": [{"id": "beta", "comment": "transmission rate"}],

 "model": [{"from": "S", "to": "I", "rate": "beta*I/N",
            "comment": "infection (N is the population size)"}], 
...
}

                 "size_t0": 1e6,

link.json

{
 "observed": [{"id": "incidence", "comment":"new cases per week",
               "definition": [{"from": "S","to": "I"}],
               "time_series_id": ["city1__CDC__prev"]}.
              ...],

 "parameter": [{"id": "rep", "comment": "reporting rate"},
               {"id": "phi", "comment": "over-dispertion of rep"}],

 "model": {"distribution": "discretized_normal",
           "mean": "rep*prop*x",
           "var": "rep*(1.0-rep)*prop*x + (rep*phi*prop*x)**2"},

 "value": {"S": {"partition_id": "variable_population", 
                 "transformation": "logit",
                 "guess": {"city1__all": 0.6, "city2__all": 0.7}}
           ...

Inference made easy

  • Create a model (will be rendered in plain C): 
    plom context.json process.json link.json -o my_model
  • Solve your problem by chaining inference methods:
    • Skip a long transient (80 000 time steps): 
      sfi | ./simul deter -T 80000
    • Run a simplex (10 000 iterations max) starting from the previously found initial conditions (-X): 
      sfi -X | ./simplex -M 10000
    • Take 1 000 000 samples with an MCMC method starting from the previous results (-B): 
      sfi -B | ./pmcmc deter -M 1e6
